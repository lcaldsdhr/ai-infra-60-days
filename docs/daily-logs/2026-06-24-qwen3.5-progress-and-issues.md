## 2. 本次会话关键收获（重要！恢复上下文用）

### 收获 1：普通 attention 的 L² 到底是什么

**L² 不是"算 1 次叠加 L² 次"**，而是**填一个 L×L 矩阵的每个格子**：

- Q·K^T 形状 [L, d] · [d, L] = [L, L] → **L² 个独立点积**
- 每个点积是 d 个数字相乘再加
- 一次 self-attention 总 FLOPs = **O(L²·d)**

> **L² 是"形状问题"（格子数），d 是"算力问题"（每个格子的精度）**

### 收获 2：d 是哪来的

**d = hidden size = 嵌入维度**，是**模型架构设计时定下的常数**：

- 每个 token 被表示成 d 维向量
- Q = x·W_Q、K = x·W_K、V = x·W_V，变换前后保持 L×d
- d 在 Q·K^T 中是"中间桥梁"，算完自动消掉
- **d 是常数（付得起），L 是变量（付不起）**

> **d 决定 token 表达精度，L 决定序列长度；GDN 故意保留 d 让 L 消失**

### 收获 3：GDN 如何利用 d²

| 利用方式 | 体现 |
|---------|------|
| **数学上** | K^T·V = d×d 关联表，d² 次内积 |
| **递推上** | S = α·S + β·(k⊗v)，每步 d² 次更新 |
| **显存上** | S 大小 = d² = 常数，不随 L 增长 |
| **推理上** | 替代 KV cache，每步 d² 次查表 |
| **并行上** | d² 摊成 d×d 矩阵用 tensor core，FLA 加速 5-10× |

**核心公式**：
```
S_t = α · S_{t-1} + β_t · (k_t ⊗ v_t)    ← 增量更新 d² 个数字
o_t = q_t · S_t                            ← 查表 d² 次乘加
```

**为什么 d² 付得起**：
- Qwen3.5 397B：d² = 4096² = 16M
- 对比 L=128K：L² = 16G
- **d²/L² = 1/1000**
